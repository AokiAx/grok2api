导入凭证

支持数组 / {accounts:[]} / ~/.grok/auth.json map。先预览，再验证写入
推荐带 refresh_token + oidc_issuer + oidc_client_id + user_id，便于自动恢复与去重
兼容格式：
1) 数组：[{"key":"...","refresh_token":"..."}]
2) 号池对象：{"accounts":[...]}
3) auth.json map：{"issuer::client::user_id":{"key":"...","refresh_token":"...","user_id":"...","expires_at":"..."}}
字段：access_token 可代替 key；expires_at / expires_in 均可；map 键里的 user_id 会自动提取。
导入会先验证上游，不会直接把坏号塞进 Ready。大批量建议先预览。
